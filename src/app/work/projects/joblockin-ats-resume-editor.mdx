# JobLockIn - ATS-Optimized Resume Editor

## Overview

JobLockIn is a web-based resume editor that enables job seekers to edit their resumes with confidence, knowing exactly when bullet points exceed ATS (Applicant Tracking System) character limits while preserving 100% formatting fidelity. Unlike traditional editors that force a trade-off between accuracy and formatting, JobLockIn provides both.

## The Problem

Job seekers face a critical challenge: they don't know if their resume bullet points will fit within ATS character limits until they submit and see truncated results.

**Existing solutions fall short:**

- **Microsoft Word**: No character limit guidance, but formatting preserved
- **Google Docs**: No tracking, export issues with DOCX
- **Online Builders**: Generic limits, lose custom formatting
- **Plain Text Editors**: Character counts available, but no formatting

**JobLockIn fills the gap:** Accurate ATS limits + 100% formatting preservation.

## Technical Architecture

**Tech Stack:** Next.js 16, React 19, TypeScript, SuperDoc, Canvas API, Zustand, Langfuse

### Split-Screen Editor

The interface features a 50/50 split-screen layout:

- **Left Panel (Form)**: Structured inputs for personal info, education, experience with real-time character counters showing percentage used (e.g., 45%, 78%, 102%)
- **Right Panel (Preview)**: SuperDoc renders pixel-perfect DOCX preview with real-time sync
- **Header**: Upload, Export DOCX, and auto-save timestamp

### Character Limit Calculation

The accuracy breakthrough comes from measuring actual rendered text:

1. **DOM Measurement**: Wait for SuperDoc to render DOCX
2. **Visual Line Count**: Measure element height / line height
3. **Tolerance-Based Rounding**: 15% threshold prevents false positives
4. **Canvas Text Width**: Measure actual character widths (handles proportional fonts)
5. **Dynamic Limits**: `limit = charsPerLine × lineCount × 0.98`

This approach handles the reality that "iii" takes less space than "www" in proportional fonts like Times New Roman.

### Bi-Directional Sync

Changes flow seamlessly between form and preview:

**Form → SuperDoc (Debounced 300ms)**
- User types in textarea
- React state updates immediately (responsive UI)
- SuperDoc sync scheduled after typing stops
- Formatting preserved using most common marks

**SuperDoc → Form (Polling 500ms)**
- Polling detects changes in document
- Extracts text using character-level position mapping
- Updates form state only if content changed
- Sync lock prevents infinite loops

## Key Features

### Accurate Character Limits
- **100% accuracy** against visual line count
- **0% false positives** (no spurious warnings)
- Color-coded feedback: Green (under 70%), Yellow (70-99%), Red (100%+)
- Percentage display with context-aware messages

### Format Preservation
- Bold, italic, underline maintained through edits
- Font sizes and styles preserved
- Bullet markers and spacing intact
- 100% fidelity in exported DOCX

### Auto-Save & Recovery
- Saves to localStorage every 30 seconds
- Automatic recovery on page reload
- Maximum 30 seconds of work lost in crash
- Fallback to sessionStorage if quota exceeded

### CRUD Operations
- Add/remove education and experience entries
- Add/remove bullets within entries
- New bullets inherit mode (most common limit)
- Minimum entry enforcement

## AI Evaluation Framework

Built a comprehensive testing system with Langfuse for quality assurance:

### 6-Dimensional Scoring

- **Factual Accuracy (30%)**: No hallucinations or fabrications
- **Length Compliance (25%)**: Fits within character limit
- **Keyword Integration (20%)**: Natural RAG keyword usage
- **Impact & Action (15%)**: Strong verbs, quantified metrics
- **Professional Tone (5%)**: Industry-appropriate language
- **Grammar & Clarity (5%)**: Correct, readable output

### Test Coverage

- **18 test cases** across 5 categories
- Technical, leadership, sales, process, edge cases
- 4 curation modes: Aggressive, Reasonable, Natural, Minimal
- **100% baseline pass rate** with 8.8/10 groundedness

### CI/CD Integration

```bash
# Run full evaluation suite
npx tsx evals/run-langfuse-evaluation.ts

# Run specific mode
npx tsx evals/run-langfuse-evaluation.ts --mode reasonable

# Debug single test
npx tsx evals/run-langfuse-evaluation.ts --test tech-backend-001 --verbose
```

- No backend required (static hosting)
- Privacy-first: data never leaves browser
- Zero infrastructure costs
- Simpler deployment

### SuperDoc Integration

- Native document editing capabilities
- Format-preserving export

### Tolerance-Based Accuracy
- 15% threshold handles sub-pixel rendering
- Eliminates false positives from rounding errors
- Matches actual visual overflow behavior
