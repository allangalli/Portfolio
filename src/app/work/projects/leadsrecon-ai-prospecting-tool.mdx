# LeadsRecon - AI Prospecting Tool for Business Development and Talent Scouting

## Overview

LeadsRecon is an end-to-end prospecting tool that captures an executive's 1st-degree professional network, enriches it on demand through pluggable data providers, and exports a vetted pipeline for both business development and talent scouting. The product pairs a Chrome extension with a cloud backend to remove the manual research bottleneck that typically separates a connection list from a usable pipeline.

## The Problem

Executives sitting on years of relationship capital, built up across hundreds or thousands of professional connections, face the same operational gap:

- Identifying who is worth a conversation requires hours of manual research per prospect
- Native export options from professional networks are limited and stripped of context
- Off-the-shelf enrichment tools don't ladder to the unique network of a specific person

The result: high-trust networks remain idle while sales and recruiting teams prospect from scratch.

## Solution

LeadsRecon automates the capture-and-enrichment workflow end-to-end:

1. **Capture**: A Chrome extension reads filtered connection lists from the executive's professional network and stores them locally and in the cloud
2. **Enrich**: On-demand enrichment workflows hydrate each contact with career history, current company, location, and wealth signals via pluggable providers
3. **Vet & Export**: A side-panel UI lets the user filter, sort, and triage contacts before exporting a CSV-ready pipeline

## Technical Architecture

**Tech Stack:** TypeScript, React 19, Chrome Manifest V3, Vite, Next.js (App Router), Drizzle ORM, Neon Postgres, Vitest

**System Design:**

- **Chrome Extension**: MV3 extension with a long-running content script for in-page capture and a Chakra-based side panel for triage. Local-first persistence in IndexedDB; cloud sync via authenticated API calls.
- **Web Backend**: Next.js App Router app on Vercel backed by Neon Postgres. Drizzle ORM with versioned migrations; Zod for request validation; cookie-based session auth.
- **Enrichment Layer**: Provider-agnostic enrichment endpoints with cost confirmation and daily quota enforcement. Cross-user contact pooling means an enriched profile is instantly available the next time anyone captures it.
- **Workflow Orchestration**: Long-running enrichment jobs are orchestrated as workflows with status polling from the side panel, so the UI stays responsive while batches process in the background.

## Key Features

### Capture
- Filtered search-results harvesting (people, company pages, self-profile)
- Human-paced pagination with randomized delays
- Soft-stop on empty results, captchas, or checkpoints
- Daily cap of 2,000 captures to keep activity within safe operational bounds

### Enrichment
- Light enrichment (name, headline, current company, location)
- Deep enrichment (full work history, education, skills, certifications)
- Profile-level caching so enrichments survive contact-deletes and re-captures
- Cross-user pooling: same profile, shared enriched data instantly
- Atomic-claim protection to prevent duplicate work on the same contact

### Triage & Export
- Resizable columns, group organization, and column visibility toggles
- "Incomplete" selector to upgrade partial Light enrichments to Deep on demand
- Cost confirmation gate for large enrichment batches
- CSV export with enrichment status and provider abstraction

## Impact

- **2,000 prospects/day** captured and triaged through a single executive's network
- **~90% reduction** in manual research time per prospect
- **On-demand enrichment** with cost transparency and quota controls
- **Dual-purpose pipeline** serving both business development and talent scouting from the same source network
- **Single-operator** workflow that previously required a dedicated researcher

## Engineering Highlights

- **318 tests** across the extension and backend (Vitest), with golden-path integration coverage
- **13 database migrations** managed through Drizzle, including rate-limit storage moved from in-memory to database-backed
- **Strict TypeScript** end-to-end, with shared types across the extension and Next.js backend
- **Cookie-based session auth** with manual approval gating for new sign-ups during the controlled rollout
